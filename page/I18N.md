# Internationalization (i18n)

## Overview

The developers.events website now supports multiple languages to make it accessible to a global audience.

## Supported Languages

- 🇬🇧 **English** (default)
- 🇫🇷 **Français** (French)
- 🇪🇸 **Español** (Spanish)
- 🇩🇪 **Deutsch** (German)
- 🇵🇹 **Português** (Portuguese)

## Features

- **Automatic language detection**: The application detects the user's browser language on first visit
- **Language persistence**: User's language preference is saved in localStorage
- **Easy language switching**: A language selector in the header allows users to change languages
- **Complete interface translation**: All UI elements, labels, buttons, and messages are translated

## Implementation

### Architecture

The internationalization system is built using:
- React Context API for state management
- JSON files for translations
- Custom `useTranslation` hook

### Usage in Components

To use translations in a component:

```jsx
import { useTranslation } from 'contexts/LanguageContext';

const MyComponent = () => {
  const { t, language, changeLanguage } = useTranslation();

  return (
    <div>
      <h1>{t('common.title')}</h1>
      <button onClick={() => changeLanguage('fr')}>
        Switch to French
      </button>
    </div>
  );
};
```

### Translation Keys

Translation keys follow a dot notation structure:
- `common.*` - Common UI elements (close, back, etc.)
- `nav.*` - Navigation items (calendar, map, list, cfps)
- `filters.*` - Filter-related text (labels, options, sorting)
- `event.*` - Event-related text (location, attendees, counts)
- `cfp.*` - CFP-related text (title, deadline, submit, until)
- `footer.*` - Footer content (madeBy, support, links)
- `addEvent.*` - Add event form (labels, errors, buttons)
- `editEvent.*` - Edit event form (title, search, submit)
- `language.*` - Language names
- `months.*` - Month names and abbreviations (january, jan, feb, etc.)
- `calendar.*` - Calendar-specific text (days, download buttons)
- `continents.*` - Continent names for region filter
- `year.*` - Year selector

Example:
```javascript
t('common.title')        // "Developer Conferences Agenda"
t('nav.calendar')        // "Calendar"
t('event.location')      // "Location"
t('months.january')      // "January" / "Janvier" / etc.
t('calendar.daysMo')     // "Mo" / "Lu" / etc.
```

### Dynamic Content with Placeholders

For translations that include dynamic content, use template strings with placeholders:

```javascript
// Translation files
{
  "event": {
    "countWithOmitted": "{count} {plural} ({omittedCount} online-only {omittedPlural} omitted)"
  },
  "calendar": {
    "yearCalendar": "{year} Calendar"
  },
  "months": {
    "monthEvents": "{month} Events:"
  }
}

// Usage in components
const text = t('event.countWithOmitted')
  .replace('{count}', count)
  .replace('{plural}', plural)
  .replace('{omittedCount}', omittedCount)
  .replace('{omittedPlural}', omittedPlural);

const calendarTitle = t('calendar.yearCalendar').replace('{year}', '2026');
const monthHeader = t('months.monthEvents').replace('{month}', translatedMonth);
```

**Important:** Use template placeholders instead of string concatenation to preserve correct word order across different languages.

## Adding a New Language

To add a new language:

1. Create a new JSON file in `page/src/locales/` (e.g., `it.json` for Italian)
2. Copy the structure from `en.json` and translate all values
3. Import the new translation file in `LanguageContext.jsx`:
   ```jsx
   import it from '../locales/it.json';
   ```
4. Add it to the translations object:
   ```jsx
   const translations = {
     en, fr, es, de, pt,
     it,  // Add new language here
   };
   ```
5. Add the language to the LANGUAGES array in `LanguageSelector.jsx`:
   ```jsx
   { code: 'it', name: 'Italiano', flag: '🇮🇹' }
   ```

## Adding New Translation Keys

When adding new UI text:

1. Add the key to all language files (`en.json`, `fr.json`, etc.)
2. Use consistent key naming (group related keys together)
3. Provide clear, contextual translations

Example:
```json
// en.json
{
  "myFeature": {
    "title": "My Feature",
    "description": "Description of my feature"
  }
}

// fr.json
{
  "myFeature": {
    "title": "Ma Fonctionnalité",
    "description": "Description de ma fonctionnalité"
  }
}
```

## Common Patterns

### Template vs Concatenation

❌ **Wrong** - String concatenation breaks word order in other languages:
```javascript
`${count} events (${omitted} online-only events omitted)`
// In French becomes: "1081 événements (omis 93 en ligne uniquement événements)"
```

✅ **Correct** - Use template placeholders:
```javascript
t('event.countWithOmitted')
  .replace('{count}', count)
  .replace('{plural}', plural)
  .replace('{omittedCount}', omittedCount)
  .replace('{omittedPlural}', omittedPlural)
// In French: "1081 événements (93 événements en ligne uniquement omis)"
```

### Conditional Fields

For forms with conditional visibility (like City/Country when "Online Event" is checked):

```javascript
{!formData.onlineEvent ? (
  <div className="form-row">
    <div className="form-group">
      <label htmlFor="city">{t('addEvent.city')} {t('addEvent.required')}</label>
      <input id="city" ... />
    </div>
  </div>
) : null}
```

## Notes

- Conference data (names, descriptions) remain in their original language
- Date formats are localized using translated month names and abbreviations
- Day abbreviations adapt to the language (Mo/Tu/We in English → Lu/Ma/Me in French)
- The HTML `lang` attribute is automatically updated when the language changes
- All error messages in forms are translated
- Template placeholders ensure correct word order across all languages
