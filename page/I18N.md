# Internationalization (i18n)

## Overview

The developers.events website now supports multiple languages to make it accessible to a global audience.

## Supported Languages

- 🇬🇧 **English** (default)
- 🇫🇷 **Français** (French)
- 🇪🇸 **Español** (Spanish)
- 🇩🇪 **Deutsch** (German)
- 🇵🇹 **Português** (Portuguese)

## Features

- **Automatic language detection**: The application detects the user's browser language on first visit
- **Language persistence**: User's language preference is saved in localStorage
- **Easy language switching**: A language selector in the header allows users to change languages
- **Complete interface translation**: All UI elements, labels, buttons, and messages are translated

## Implementation

### Architecture

The internationalization system is built using:
- React Context API for state management
- JSON files for translations
- Custom `useTranslation` hook

### File Structure

```
page/src/
├── contexts/
│   └── LanguageContext.jsx          # Language context and hook
├── locales/
│   ├── en.json                      # English translations
│   ├── fr.json                      # French translations
│   ├── es.json                      # Spanish translations
│   ├── de.json                      # German translations
│   └── pt.json                      # Portuguese translations
└── components/
    └── LanguageSelector/
        ├── LanguageSelector.jsx      # Language selector component
        └── LanguageSelector.css      # Styles
```

### Usage in Components

To use translations in a component:

```jsx
import { useTranslation } from 'contexts/LanguageContext';

const MyComponent = () => {
  const { t, language, changeLanguage } = useTranslation();

  return (
    <div>
      <h1>{t('common.title')}</h1>
      <button onClick={() => changeLanguage('fr')}>
        Switch to French
      </button>
    </div>
  );
};
```

### Translation Keys

Translation keys follow a dot notation structure:
- `common.*` - Common UI elements
- `nav.*` - Navigation items
- `filters.*` - Filter-related text
- `event.*` - Event-related text
- `cfp.*` - CFP-related text
- `footer.*` - Footer content
- `language.*` - Language names

Example:
```javascript
t('common.title')        // "Developer Conferences Agenda"
t('nav.calendar')        // "Calendar"
t('event.location')      // "Location"
```

## Adding a New Language

To add a new language:

1. Create a new JSON file in `page/src/locales/` (e.g., `it.json` for Italian)
2. Copy the structure from `en.json` and translate all values
3. Import the new translation file in `LanguageContext.jsx`:
   ```jsx
   import it from '../locales/it.json';
   ```
4. Add it to the translations object:
   ```jsx
   const translations = {
     en, fr, es, de, pt,
     it,  // Add new language here
   };
   ```
5. Add the language to the LANGUAGES array in `LanguageSelector.jsx`:
   ```jsx
   { code: 'it', name: 'Italiano', flag: '🇮🇹' }
   ```

## Adding New Translation Keys

When adding new UI text:

1. Add the key to all language files (`en.json`, `fr.json`, etc.)
2. Use consistent key naming (group related keys together)
3. Provide clear, contextual translations

Example:
```json
// en.json
{
  "myFeature": {
    "title": "My Feature",
    "description": "Description of my feature"
  }
}

// fr.json
{
  "myFeature": {
    "title": "Ma Fonctionnalité",
    "description": "Description de ma fonctionnalité"
  }
}
```

## Notes

- Conference data (names, descriptions) remain in their original language
- Date formatting can be localized using the browser's Intl API if needed
- The HTML `lang` attribute is automatically updated when the language changes
